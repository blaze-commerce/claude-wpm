```
│                        FILE MAPPING WORKFLOW                            │

│                                                                         │
│  1. Developer adds/removes files → Pushes to main branch               │
│                           ↓                                             │
│  2. GitHub Actions triggers update-mapping.yml workflow                 │
│                           ↓                                             │
│  3. Workflow runs `find` command (built-in Unix command)               │
│     - Discovers ALL files in repository                                 │
│     - Labels each file: [WPM], [WPM-QA], or [REPO]                        │
│                           ↓                                             │

│  5. On release: verify-deploy-zip.sh reads [WPM] files from here       │
│     - This ensures verification matches actual file inventory           │
│     - Single source of truth = no manual sync needed                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```
